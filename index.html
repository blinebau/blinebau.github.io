Projects

Board Game Server

Description

Type: Coursework

Tools Used: Java, JavaFX, Java Socket Networking

This project was apart of the coursework for a software design class at UCI. It was completed as a group assignment for the class and required groups to build a game server platform that multiple users could connect to and play against eachother. The platform we built supports Tic-Tac-Toe, Checkers and Chess, and supports multiple client connection for a multide of hosted and active games. My role in the project was creating the infrastructure for the platform by implementing the client-server programs necessary for network communication. The choice of language for the project was Java so, I utilized the Java Socket library for custom networking. This allowed for messages whether it be connection requests, commands or moves for a game to be sent through a two-way communication path between the client(s) and server programs. The server would be the recipient of messages and would make decisions or delegate moves to a client depending on the content of the request. In addition, I also was responsible for implementing GUI for the client and server applications, this included: server login and connection logging interfaces, client game lobby and manipulation of the frame resulting from click events. The result of the project was a functional client-server (multi-threaded) application that multiple clients on the same host machine can play against eachother on a handful of supported board games. The project could be adapted to allow external clients to run client programs and connect to a host machine that is running the server program.

Exergame

Description

Type: Coursework, Client Project

Tools Used: Java, Spring Framework , MongoDB

A large-scale project that was a part of a project course that focused on human computer interaction. The details of this project were outside the scope of this course, and had heavy implementation elements involved. The project was worked on as a group with an assigned client that was met on a weekly basis to discuss project specification, current builds, issues, user requirements and etc. The project required the prototyping of a web application that would be used as a patient administration system for doctors and their leukemia patients. The application was responsible for visualizing patient data, tracking patient progress and adjusting patient histories and prescriptions. The application had to be secure as sensitive medical information between it and a medical database. My role in the project was implementing the RESTful API endpoints, data layer and security for the application. The Spring Framework was heavily used in the project to simplify Java web development requirements, in addition it offers a wide variety of projects that suited the application requirements. I used Spring Data for ease of data access and modeling as well as its supporting sub-project which integrates well with MongoDB datastores, our primary datastore in this project. I used Spring Security and its supporting authentication processes to provide secure accessing, request filter, data response and user verification for the application.
